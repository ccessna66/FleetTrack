CONNECT GOOGLE SHEETS

One-time setup • Your data stays in your Google account
1
Go to Google Cloud Console

Open console.cloud.google.com and sign in with your Google account.

2
Create a new project

Click the project dropdown at the top → New Project. Name it anything (e.g. FleetTrack). Click Create.

3
Enable the Google Sheets API

In the left menu go to APIs & Services → Library. Search for Google Sheets API and click Enable.

4
Create an OAuth 2.0 Client ID

Go to APIs & Services → Credentials → Create Credentials → OAuth client ID.
• Application type: Web application
• Name: FleetTrack
• Authorized JavaScript origins: add https://yourusername.github.io (replace with your actual GitHub username)
Click Create and copy the Client ID (ends in .apps.googleusercontent.com).

5
Configure OAuth consent screen (if prompted)

Go to APIs & Services → OAuth consent screen. Choose External, fill in the app name (FleetTrack), your email, and save. Add yourself as a Test user.

6
Host on GitHub Pages

• Create a free account at github.com
• Create a new public repository (e.g. fleettrack)
• Upload this HTML file, renamed to index.html
• Go to Settings → Pages → Branch: main → Save
• Your app will be live at https://yourusername.github.io/fleettrack/
• On your phone, open that URL in Safari/Chrome and use Add to Home Screen to install it like an app

7
Enter your Client ID below

Paste the Client ID you copied in Step 4. It will be saved in your browser only.

8
Already have a FleetTrack spreadsheet? Pin its ID (optional — for syncing across devices)

Open your existing FleetTrack spreadsheet in Google Sheets. Copy the long ID from the URL:
docs.google.com/spreadsheets/d/THIS_PART/edit

Paste it below so this device always connects to the same sheet as your other devices.

🔒 Privacy: Your Client ID and all vehicle/expense data stay entirely within your Google account. FleetTrack communicates directly between your browser and Google's API — no data ever passes through any third-party server.
